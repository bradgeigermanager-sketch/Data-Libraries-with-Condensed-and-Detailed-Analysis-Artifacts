To elevate this ecosystem from a purely theoretical logic graph into a functional repository for reusable machine-readable maps, templates, and structural schemas, the core architecture must be expanded to polymorphically handle different artifact types.
By categorizing records at the database layer, the system can natively store, test, and retrieve generalized, fillable-form code with explicit usage notations, seamlessly integrating these assets alongside the logical concepts that govern them.
Here are the structural upgrades required to support this unified library.
### 1. Database Upgrade: Polymorphic Artifact Support (SQL)
The core table must be modified to distinguish between an abstract logic node and an actionable code template. This allows the backend to route the data differently based on its utility.
```sql
/*
  NOTATION: Database Schema Expansion Map.
  USE: Execute against existing PostgreSQL database to add typing and language classification to the core table.
*/

-- Step 1: Add classification columns
ALTER TABLE LogicArtifact 
ADD COLUMN artifact_category VARCHAR(50) DEFAULT 'logic_node', -- e.g., 'logic_node', 'machine_map', 'fillable_schema'
ADD COLUMN syntax_language VARCHAR(50) DEFAULT 'text'; -- e.g., 'python', 'sql', 'json_ld', 'html'

-- Step 2: Index the new category for rapid filtering in the Browse UI
CREATE INDEX idx_artifact_category ON LogicArtifact(artifact_category);

-- Step 3: Update Immutability Triggers (from previous implementation)
-- Ensure the new columns are included in the immutability enforcement rules so that schemas cannot be mutated without a 'supersedes' version edge.
CREATE OR REPLACE FUNCTION prevent_artifact_mutation()
RETURNS TRIGGER AS $$
BEGIN
    IF NEW.condensed_logic <> OLD.condensed_logic OR 
       NEW.verbose_desc <> OLD.verbose_desc OR 
       NEW.foundational_data <> OLD.foundational_data OR
       NEW.artifact_category <> OLD.artifact_category OR
       NEW.syntax_language <> OLD.syntax_language THEN
        RAISE EXCEPTION 'Immutability Violation: Core artifacts, including generic templates, cannot be mutated. Author a new version and link it via a "supersedes" edge.';
    END IF;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

```
