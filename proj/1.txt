2. Backend package structure

`text
backend/
  logic_core/
    init.py
    models.py          # LogicArtifact, LogicRelationship dataclasses/ORM
    relationships.py   # relationship types, helpers
    versioning.py      # supersedes, deprecated flags
    contradictions.py  # detection + classification

  db/
    init.py
    connection.py      # DB URL, pool, session
    schema.sql         # init schema
    immutability.sql   # triggers
    repository.py      # CRUD + graph queries (getlogicartifact, directory index, resolver)

  ingestion/
    init.py
    engine.py          # LogicIngestionEngine
    normalize.py
    extract.py
    package_jsonld.py
    state_tracker.py
    resolver.py        # title → UUID, autocomplete helpers

  wikipedia/
    init.py
    client.py          # WikipediaClient
    category_harvester.py
    wiki_extract.py    # summary/infobox → ingestion-ready structures

  api/
    init.py
    main.py            # FastAPI/Flask app
    routes_logic.py    # /api/logic, /ui/* renderers
    routes_admin.py    # /api/admin/ingest, versioning
    routes_directory.py# /api/directory

5. How responsibilities split

- logic_core: pure domain logic, no IO.
- db: all SQL, connections, and repositories.
- ingestion: normalization, extraction, packaging, state tracking, resolver.
- wikipedia: all MediaWiki-specific logic.
- api: HTTP surface only—thin layer over core + db + ingestion.
- frontend: static assets, no backend logic.
- deploy: everything runtime/infra.
