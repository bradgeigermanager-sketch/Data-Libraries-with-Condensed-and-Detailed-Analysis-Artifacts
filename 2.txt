### Frame 1: The Directory Map (Browse Frame)
The directory frame serves as the persistent navigation index. It queries the LogicArtifact core nodes and generates a list of triggers. When an item is selected, it routes the command through the parent window to update the other three context frames simultaneously.
**Directory Frame Template (browse.html)**
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <style>
        body { font-family: monospace; color: #cccccc; padding: 10px; }
        ul { list-style-type: none; padding: 0; }
        li { padding: 8px 0; border-bottom: 1px solid #444; cursor: pointer; }
        li:hover { color: #ffffff; background-color: #333333; }
    </style>
</head>
<body>
    <h3>Library Directory</h3>
    <input type="text" placeholder="Search logic nodes..." style="width: 90%; margin-bottom: 10px;">
    
    <ul id="logic-list">
        <li onclick="requestArtifactLoad('[UUID_1]')">[Logic Title 1]</li>
        <li onclick="requestArtifactLoad('[UUID_2]')">[Logic Title 2]</li>
        <li onclick="requestArtifactLoad('[UUID_N]')">[Logic Title N]</li>
    </ul>

    <script>
        /*
          NOTATION: Global state trigger.
          USE: Broadcasts the selected UUID to the main wrapper index.html.
        */
        function requestArtifactLoad(id) {
            window.parent.postMessage({ type: 'LOAD_ARTIFACT', id: id }, '*');
        }
    </script>
</body>
</html>

```
